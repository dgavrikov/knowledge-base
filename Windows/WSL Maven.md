### Инструкция по установке Maven (версии 3.9.9 или выше) в WSL (Ubuntu или Debian)

Эта инструкция универсальна для WSL на базе Ubuntu или Debian. В стандартных репозиториях Maven часто устаревший (3.6.x–3.8.x), поэтому для версии 3.9.9+ рекомендуется SDKMAN или ручная установка. Если у вас Ubuntu/Debian с обновлёнными репозиториями, версия может быть доступна через apt.

---

#### 1. Обновите систему и проверьте доступную версию Maven
```bash
sudo apt update
apt show maven
```
- Посмотрите на версию в выводе (поле "Version").
- Если версия 3.9.9 или выше, переходите к шагу 2.
- Если ниже или отсутствует, переходите к шагу 3 (SDKMAN).

#### 2. Установка через apt (если версия подходит)
```bash
sudo apt install maven
```
Переходите к шагу 4.

#### 3. Установка через SDKMAN (рекомендуется для свежей версии)
SDKMAN позволяет установить последнюю версию Maven (3.9.9+).
```bash
# Если SDKMAN ещё не установлен (из JDK-инструкции), установите:
curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"

# Установите Maven
sdk install maven
```
SDKMAN автоматически установит последнюю версию и настроит `PATH`.

#### 4. Настройка переменных окружения (если устанавливали через apt)
Если устанавливали через apt (шаг 2), Maven обычно доступен глобально, но убедитесь, что `PATH` корректен. Добавьте в `~/.bashrc` (если нужно):
```bash
export PATH=/usr/bin:$PATH  # Maven обычно здесь
```
Примените:
```bash
source ~/.bashrc
```

#### 5. Проверка установки
```bash
mvn -v
```
Должно показать версию 3.9.9 или выше (например, "Apache Maven 3.9.9").

---

#### Примечания
- **Если apt устанавливает старую версию**: Используйте SDKMAN для обновления.
- **Ручная установка (альтернатива SDKMAN)**: Скачайте архив с [maven.apache.org](https://maven.apache.org/download.cgi), распакуйте в `~/maven` и добавьте в `~/.bashrc`:
  ```bash
  export MAVEN_HOME=~/maven/apache-maven-3.9.9
  export PATH=$MAVEN_HOME/bin:$PATH
  ```
- **Версия системы**: Проверьте `lsb_release -a`.
- **Совместно с JDK**: Убедитесь, что JDK 21 установлен (по предыдущей инструкции), иначе Maven не будет работать.

Эта инструкция объединяет подходы для удобства. Если нужны уточнения или помощь с ошибками — скажите!

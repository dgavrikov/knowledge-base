### Включение и установка WSL (Windows Subsystem for Linux)

WSL позволяет запускать Linux-дистрибутивы в Windows без виртуальной машины. Требования: Windows 10 (версия 2004 или новее) или Windows 11, включённая виртуализация в BIOS/UEFI. Запускайте команды от имени администратора (в PowerShell или командной строке).

#### 1. Включение компонентов Windows
Выполните эти команды по очереди для активации WSL и платформы виртуальных машин:

```
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
```
```
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```

**Важно:** После выполнения перезагрузите компьютер, чтобы изменения вступили в силу.

#### 2. Установка WSL
Установите WSL и дистрибутив по умолчанию (Ubuntu):

```
wsl --install
```

Если хотите сразу установить конкретный дистрибутив (например, Debian), используйте:

```
wsl --install -d Debian
```

Это также установит ядро WSL, если его нет.

#### 3. Настройка версии WSL
WSL2 рекомендуется для лучшей производительности. Установите версию по умолчанию:

```
wsl.exe --set-default-version 2
```

Чтобы изменить версию для конкретного дистрибутива (например, для Debian):

```
wsl.exe --set-version Debian 2
```

#### 4. Работа с дистрибутивами
- **Просмотр доступных дистрибутивов для скачивания:**

  ```
  wsl.exe --list --online
  ```

- **Просмотр установленных дистрибутивов и их версий WSL:**

  ```
  wsl.exe --list --verbose
  ```

  Или сокращённо: `wsl.exe -l -v`

- **Установка дополнительного дистрибутива (пример: Ubuntu):**

  ```
  wsl.exe --install -d Ubuntu
  ```

- **Запуск дистрибутива:**
  После установки введите `wsl` или `wsl -d <DistroName>` (например, `wsl -d Debian`). При первом запуске настройте имя пользователя и пароль Linux.

- **Выход из WSL:** В терминале введите `exit`.

- **Удаление дистрибутива:**

  ```
  wsl.exe --unregister <DistroName>
  ```

#### 5. Дополнительные настройки и советы
- **Обновление WSL:** Регулярно обновляйте для исправления уязвимостей:

  ```
  wsl --update
  ```

- **Доступ к файлам:**
  - Из WSL: Доступ к дискам Windows через `/mnt/c/` (например, `cd /mnt/c/Users`).
  - Из Windows: Файлы WSL находятся в `\\wsl$\<DistroName>` (откройте в Проводнике).

- **Проверка статуса:** Если возникают проблемы, проверьте статус WSL:

  ```
  wsl --status
  ```

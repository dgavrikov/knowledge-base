# Плагины для VS Code

## Основные
Плагины на каждый день.

### Material Icon Theme
<img src="https://pkief.gallerycdn.vsassets.io/extensions/pkief/material-icon-theme/5.27.0/1757312653977/Microsoft.VisualStudio.Services.Icons.Default" alt="Git Graph Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=PKief.material-icon-theme

**Описание:** Этот плагин заменяет стандартные иконки файлов и папок в проводнике VS Code на стильные иконки в стиле Material Design. Поддерживает сотни расширений файлов (от .js до .py), с возможностью настройки через настройки VS Code (например, изменение цвета папок или добавление кастомных иконок). Улучшает визуальную навигацию, делает интерфейс более современным и приятным для глаз. Легковесный, не влияет на производительность. Идеально для тех, кто хочет персонализировать рабочее пространство без лишних усилий.

### Docker
<img src="https://ms-azuretools.gallerycdn.vsassets.io/extensions/ms-azuretools/vscode-docker/2.0.0/1748459272971/Microsoft.VisualStudio.Services.Icons.Default" alt="Git Graph Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=ms-azuretools.vscode-docker

**Описание:** Официальный плагин от Microsoft для интеграции Docker с VS Code. Позволяет управлять контейнерами и образами прямо из редактора: запускать, останавливать, просматривать логи, строить и отлаживать приложения в контейнерах. Поддерживает подсветку синтаксиса для Dockerfile и docker-compose.yml, автодополнение команд и интеграцию с Azure. Упрощает разработку и развертывание контейнеризованных приложений, особенно полезен для DevOps и микросервисных архитектур. Требует установленного Docker Desktop.

### Git Graph
<img src="https://mhutchie.gallerycdn.vsassets.io/extensions/mhutchie/git-graph/1.30.0/1617594001998/Microsoft.VisualStudio.Services.Icons.Default" alt="Git Graph Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=mhutchie.git-graph

**Описание:** Мощный инструмент для визуализации истории Git-репозитория в виде интерактивного графа. Отображает коммиты, ветви, слияния и теги в удобном интерфейсе прямо в VS Code. Позволяет просматривать детали коммитов, сравнивать изменения, выполнять операции (cherry-pick, revert) и интегрируется с командной строкой Git. Идеален для управления сложными проектами с множеством веток, улучшает понимание структуры репозитория и упрощает отладку истории изменений. Не требует дополнительных настроек, работает с любым Git-репо.

### Git History Diff
<img src="https://huizhou.gallerycdn.vsassets.io/extensions/huizhou/githd/2.5.6/1739948104239/Microsoft.VisualStudio.Services.Icons.Default" alt="Git Graph Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=huizhou.githd

**Описание:** Плагин для просмотра истории изменений файлов в Git-репозитории прямо в VS Code. Позволяет сравнивать версии файлов, просматривать diff между коммитами, ветками или с рабочей директорией. Интегрируется с проводником, поддерживает поиск по истории и визуализацию изменений. Полезен для отладки кода, понимания эволюции файлов и ревью изменений. Работает без дополнительных зависимостей, просто устанавливается и готов к использованию.

### WSL
<img src ="https://ms-vscode-remote.gallerycdn.vsassets.io/extensions/ms-vscode-remote/remote-wsl/0.104.3/1759355620566/Microsoft.VisualStudio.Services.Icons.Default" alt="Git Graph Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-wsl

**Описание:** Официальный плагин Microsoft для интеграции VS Code с Windows Subsystem for Linux (WSL). Позволяет открывать проекты в WSL прямо из VS Code на Windows, редактировать файлы в Linux-среде с полным доступом к терминалу и инструментам. Поддерживает отладку, Git и расширения внутри WSL, обеспечивая бесшовную разработку кросс-платформенных приложений. Требует установленного WSL и VS Code Remote Development. Упрощает работу с Linux-проектами на Windows без виртуальных машин.

### TODO Highlight
<img src="https://wayou.gallerycdn.vsassets.io/extensions/wayou/vscode-todo-highlight/1.0.5/1635478170130/Microsoft.VisualStudio.Services.Icons.Default" alt="Git Graph Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=wayou.vscode-todo-highlight

**Описание:** Плагин для выделения и поиска комментариев с ключевыми словами, такими как TODO, FIXME, NOTE, в коде. Помогает разработчикам отслеживать задачи, багфиксы и заметки в проекте, улучшая организацию работы. Легковесный и не требует дополнительных зависимостей.

Поддерживает подсветку синтаксиса и автодополнение. Идеален для разработчиков API, упрощает работу с документацией REST API и микросервисов. Не требует дополнительных инструментов.

## Работа с Markdown

### Markdown All in One
<img src="https://yzhang.gallerycdn.vsassets.io/extensions/yzhang/markdown-all-in-one/3.6.3/1741534224980/Microsoft.VisualStudio.Services.Icons.Default" alt="Markdown All in One Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=yzhang.markdown-all-in-one

**Описание:** Комплексный плагин для работы с Markdown в VS Code, объединяющий множество полезных функций. Включающее автодополнение, клавиатурные сокращения (например, для жирного текста, ссылок), встроенный превью, навигацию по заголовкам, экспорт в HTML/PDF, сортировку таблиц и поддержку GitHub Flavored Markdown. Упрощает редактирование и форматирование документов, интегрируется с VS Code без дополнительных настроек. Идеально для писателей, разработчиков и тех, кто часто работает с документацией. Регулярно обновляется, поддерживает темы и кастомизацию через настройки. Может конфликтовать с другими Markdown-плагинами, так что используйте осторожно.

### Markdownlint
<img src="https://davidanson.gallerycdn.vsassets.io/extensions/davidanson/vscode-markdownlint/0.60.0/1748308772092/Microsoft.VisualStudio.Services.Icons.Default" alt="Markdownlint Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=DavidAnson.vscode-markdownlint

**Описание:** Плагин для линтинга Markdown-файлов в VS Code, улучшающий качество и一致ность документации. Интегрирует библиотеку markdownlint в VS Code для проверки Markdown-файлов на ошибки форматирования, несогласованности и лучших практик (например, правильное использование заголовков, списков, ссылок и пустых строк). Поддерживает конфигурацию через .markdownlintrc файлы или настройки VS Code, показывает предупреждения и ошибки в реальном времени с предложениями исправлений. Полезен для разработчиков, писателей и команд, работающих с документацией (README, вики, блоги). Работает без дополнительных зависимостей, регулярно обновляется и совместим с GitHub Flavored Markdown. Легковесный, но может быть настроен для игнорирования определенных правил в проектах.

## Rust
Плагины для разработки на Rust, включая анализ кода, отладку и управление проектами.

### Rust Analyzer
<img src="https://rust-lang.gallerycdn.vsassets.io/extensions/rust-lang/rust-analyzer/0.4.2640/1760057523274/Microsoft.VisualStudio.Services.Icons.Default" alt="Rust Analyzer Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=rust-lang.rust-analyzer

**Описание:** Официальный Language Server для Rust, предоставляет продвинутую поддержку: автодополнение, подсветку ошибок, рефакторинг, навигацию по коду и интеграцию с Cargo. Работает на базе Rust toolchain, улучшает производительность и точность анализа. Идеален для быстрой разработки, требует установки Rust (через rustup). Легковесный и регулярно обновляется.

### CodeLLDB
<img src="https://vadimcn.gallerycdn.vsassets.io/extensions/vadimcn/vscode-lldb/1.11.6/1759815798834/Microsoft.VisualStudio.Services.Icons.Default" alt="CodeLLDB Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=vadimcn.vscode-lldb

**Описание:** Мощный отладчик для Rust и других языков на базе LLDB. Позволяет ставить точки останова, инспектировать переменные, пошагово выполнять код и интегрируется с VS Code. Поддерживает Rust-фичи вроде ownership и borrowing. Упрощает отладку, особенно в сложных сценариях, и работает с Rust-проектами без дополнительных настроек.

### Crates
<img src="https://serayuzgur.gallerycdn.vsassets.io/extensions/serayuzgur/crates/0.6.7/1719591384441/Microsoft.VisualStudio.Services.Icons.Default" alt="Crates Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=serayuzgur.crates

**Описание:** Помогает управлять зависимостями в Cargo.toml: показывает версии пакетов, обновления, документацию и позволяет быстро добавлять/обновлять crates. Интегрируется с crates.io, подсвечивает устаревшие зависимости и предлагает автодополнение. Улучшает управление проектами, экономит время на ручной поиск и требует установленного Cargo.

Поддерживает специфику Rust-проектов, исправляет ошибки и улучшает читаемость. Легковесное, не требует дополнительных инструментов и идеально для редактирования конфигураций в Rust.

### Rust Syntax
<img src="https://dustypomerleau.gallerycdn.vsassets.io/extensions/dustypomerleau/rust-syntax/0.6.1/1657445670505/Microsoft.VisualStudio.Services.Icons.Default" alt="Rust Syntax Icon" width="128" height="128">
Ссылка: https://marketplace.visualstudio.com/items?itemName=dustypomerleau.rust-syntax

Описание: Простой плагин для подсветки синтаксиса Rust в VS Code. Предоставляет базовое выделение ключевых слов, типов, функций и комментариев, улучшая читаемость кода без установки полного toolchain. Не включает анализ ошибок, автодополнение или отладку — для этого нужен Rust Analyzer. Подходит для быстрого просмотра или редактирования файлов, если вы не хотите тяжелые инструменты. Легковесный, но устаревший по сравнению с Rust Analyzer; рекомендуется только как дополнение или для минималистичных сред. Последние обновления редкие, так что проверяйте совместимость с новыми версиями Rust.

### Pylint
<img src="https://ms-python.gallerycdn.vsassets.io/extensions/ms-python/pylint/2025.3.12271016/1755253563492/Microsoft.VisualStudio.Services.Icons.Default" alt="Pylint Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=ms-python.pylint

**Описание:** Расширение для линтинга Python-кода с помощью Pylint, помогающее выявлять ошибки, улучшать стиль и следовать лучшим практикам (например, проверка на неиспользуемые переменные, длинные строки и PEP 8). Показывает предупреждения и ошибки в реальном времени с предложениями исправлений. Полезно для новичков и команд, стремящихся к чистому коду. Интегрируется с Python-расширением, поддерживает конфигурацию через pylintrc-файлы. Легковесное, но может замедлить работу при больших файлах; отключайте для проектов с другими линтерами.

### Black Formatter
<img src="https://ms-python.gallerycdn.vsassets.io/extensions/ms-python/black-formatter/2025.3.11831009/1751451431219/Microsoft.VisualStudio.Services.Icons.Default" alt="Black Formatter Icon" width="128" height="128">

**Ссылка:** https://marketplace.visualstudio.com/items?itemName=ms-python.black-formatter

**Описание:** Автоматический форматтер для Python-кода на основе Black, обеспечивающий единообразное форматирование по стандартам PEP 8 (например, правильные отступы, длины строк и кавычки). Форматирует код при сохранении или по команде, без необходимости ручных правок. Идеально для команд, где стиль кода критичен, и для экономии времени. Интегрируется с VS Code, поддерживает конфигурацию через pyproject.toml. Может конфликтовать с другими форматтерами; настройте как дефолтный в настройках Python.

### Инструкция по настройке SSH-ключа для GitHub

Эта инструкция поможет сгенерировать SSH-ключ и добавить его для аутентификации на GitHub. Предполагается, что ты используешь Linux, macOS или WSL/Windows с Git Bash.

#### 1. Генерация SSH-ключа с помощью ssh-keygen
1. Открой терминал (в WSL или Git Bash).
2. Запусти команду генерации ключа. Рекомендуется тип Ed25519 (безопасный и быстрый):
   ```
   ssh-keygen -t ed25519 -C "your_email@example.com"
   ```
   - Замени `"your_email@example.com"` на свой email, привязанный к GitHub.
   - Нажми Enter, чтобы принять путь по умолчанию (`~/.ssh/id_ed25519`).
   - Опционально: введи пароль для защиты ключа (рекомендуется для безопасности).
3. После генерации в `~/.ssh/` появятся файлы: `id_ed25519` (приватный ключ) и `id_ed25519.pub` (публичный ключ).

#### 2. Добавление ключа
1. **Копирование в каталог ~/.ssh** (если ключи не там):
   - Если ключи сгенерированы в другом месте, скопируй их в `~/.ssh/` (стандартный каталог):
     ```
     cp path/to/id_ed25519 ~/.ssh/
     cp path/to/id_ed25519.pub ~/.ssh/
     ```
   - Установи правильные права: `chmod 600 ~/.ssh/id_ed25519` (приватный) и `chmod 644 ~/.ssh/id_ed25519.pub` (публичный).

2. **Добавление ключа в SSH-агент:**
   - Запусти SSH-агент: `eval "$(ssh-agent -s)"`.
   - Добавь приватный ключ: `ssh-add ~/.ssh/id_ed25519`.
   - Если ключ защищён паролем, введи его.
   - Проверь: `ssh-add -l` (должен показать твой ключ).

3. **Регистрация в GitHub:**
   - Скопируй содержимое публичного ключа: `cat ~/.ssh/id_ed25519.pub`.
   - Зайди в GitHub: Settings > SSH and GPG keys > New SSH key.
   - Вставь скопированный ключ, дай название (например, "My SSH Key") и сохрани.
   - Проверь подключение: `ssh -T git@github.com` (должен сказать "successfully authenticated").

Теперь можешь клонировать репозитории по SSH: `git clone git@github.com:username/repo.git`. Если проблемы — проверь логи с `ssh -v git@github.com`.
